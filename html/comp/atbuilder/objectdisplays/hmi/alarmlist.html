Alarmlist

The alarmlist is an advanced control that displays the current active alarms of the system and defined alarm conditions in form of a table (including filter possibilities).

Views

There are several views which can be preset through display parameters:

  • alarms – All triggered alarms are listed here.

  • all – All triggered alarms and all defined alarm conditions are listed here.

  • shelved – All shelved alarms and alarm conditions are listed here.

  • suppressed – All suppressed alarms and alarm conditions are listed here.

  • disabled – All disabled alarms and alarm conditions are listed here.

The views can be selected in the head of the atvise standard visualization or through the "Open alarm list" button.

Priority

The (column) priority defined in the global list is used to display different columns on different devices (screen sizes). Columns of priority 1 are displayed on every device, columns of priority 2 and higher are hidden on small displays (smartphones), columns of priority 3 and higher are only visible on bigger displays (desktop computers).

Filter

On top of the alarm list display you can find a set of filter elements. After changing the input elements and clicking the "Apply filter" button, the new filter is applied to the alarmlist.

  • Min. Priority – "Greater equal" filter by priority (defined at the alarm category).

  • Display – Filter by the attribute "display" that can be defined in the alarm configuration.

  • Group – Filter for alarms of one or more "alarm groups". Multiple alarm groups can be entered comma- or semicolon-seperated.

  • Alarm source – Filter by the data source of alarms.

  • Alarm condition – Filter by the node of the alarm condition.

  • Ackn. state – Filter by ALL or one of the alarm states:

    • ACTIVE!

    • ACTIVE! unconf.

Alarms

Each alarm can be acknowledged.

Confirmable alarms can be confirmed after acknowledging.

A link to the corresponding display will be available if the display was specified in the configuration.

For retrigger-alarms each triggering state change causes an distinct alarm in the table.

The alarm category "Acknowledge active and inactive" with explicit acknowledgement is displayed as a pair of alarms with identical background color. The inactive alarm state is shown on top of the active alarm state. Before the inactive state can be acknowledged the acknowledgement of the active state is required.

For historized alarms please use the "history list" display.

Actions

  • acknowledge – An alarm is acknowledged.

  • confirm – An acknowledged alarm is confirmed.

  • acknowledge with comment – An alarm is acknowledged via a pop up dialog which allows to pass a comment.

  • confirm with comment – An acknowledged alarm is confirmed via a pop up dialog which allows to pass a comment.

  • shelve – Alarm or alarm condition is being shelved. The user cannot see alarms and conditions in the regular alarm list any longer.

    • one shot shelve – This action can only be applied to alarms and not to alarm conditions. The alarm is shelved until it is unshelved or the maximum shelving time set in the MaxTimeShelved variable is reached.

    • timed shelve – This action can be applied to alarms and alarm conditions. The alarm or alarm condition is shelved for a time period set by the user in a pop up dialog.

  • unsuppress – Alarm or alarm condition is being unsuppressed. Suppressing happens internally on the server. Suppressed alarms are not shown in the regular alarm list.

  • disable – Alarm condition and associated active alarms are being disabled. No further alarms and events are generated.

Detail table

Further details of the alarms and conditions can be displayed in a foldable detail table.

The fields shown in the detail table can be configured through a global list whose address can be selected in the display parameters.

If new alarms appear in the alarm list, the rows of the shown alarms can jump to different positions unfavorably and display the detail table on a different position. To fix the detail table or certain rows, the favored rows can be marked using the checkboxes on the left. After that you can use the filter to view only the marked rows.

Security:

  • right for acknowledge – If the right for acknowledge is being set, a user can only accept an alarm if he has the right for acknowledge. In the case that a user does not have the right for acknowledge, the according symbols and links will be inactive. If no right for acknowledge is specified, alarms can be accepted by any user.

  • right for confirm – If the right for confirm is being set, a user can only confirm an alarm if he has the right for confirm. In the case that a user does not have the right for confirm, the according symbols and links will be inactive. If no right for confirm is specified, alarms can be confirmed by any user.

  • right for (un)shelve – If the right for (un)shelve is being set, a user can only shelve or unshelve alarms and conditions if he has the right for (un)shelve. In the case that a user does not have the right for (un)shelve, the according symbols and links will be inactive. If no right for (un)shelve is specified, alarms or conditions can be shelved or unshelved by any user.

  • right for unsuppress – If the right for unsuppress is being set, a user can only unsuppress an alarm condition if he has the right for unsuppress. In the case that a user does not have the right for unsuppress, the according symbols and links will be inactive. If no right for unsuppress is specified, alarm conditions can be unsuppressed by any user.

  • right for disable/enable – If the right for disable/enable is being set, a user can only disable or enable alarm conditions if he has the right for disable/enable. In the case that a user does not have the right for disable/enable, the according symbols and links will be inactive. If no right for disable/enable is specified, alarm conditions can be enabled or disabled by any user.

  • right for CSV export – If the right for CSV export is being set, a user can only use the CSV export an alarm if he has the right for CSV export. In the case that a user does not have the right for CSV export, the according links will be inactive. If no right for CSV export is specified, CSV export can be carried out by any user.
